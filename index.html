ARDUINO

TÍTULO DE LA PRÁCTICA
"Utilización de teclado 4x4, pantalla lcd "

Crea un sistema de inicio de sesión, si la contraseña es correcta mostrar en el lcd CORRECTO y encender un led, si es incorrecto encender una alarma y mostrar INCORRECTO en el lcd

Introducción

El desarrollo de sistemas de control de acceso es fundamental en la seguridad de dispositivos electrónicos. El uso de un teclado matricial 4x4 permite ingresar contraseñas de manera rápida y precisa. La pantalla LCD ofrece retroalimentación visual inmediata al usuario, mostrando mensajes claros según la validez de la contraseña. Además, la combinación de indicadores como LEDs y buzzer proporciona alertas visuales y sonoras que mejoran la interacción y seguridad. Este proyecto integra hardware y software, fomentando la comprensión de la programación de microcontroladores y el manejo de periféricos. Con ello, se logra un sistema confiable y educativo para prácticas de electrónica y programación.

Objetivo General

Desarrollar un sistema de control de acceso utilizando un teclado matricial y una pantalla LCD, que permita validar la contraseña ingresada y mostrar mensajes visuales y sonoros según el resultado.

Objetivos Específicos

  • Configurar correctamente la pantalla LCD para mostrar mensajes de forma clara y centrada.
  • Implementar la lectura del teclado matricial para capturar la contraseña ingresada por el usuario.
  • Validar la contraseña y generar respuestas visuales (LED) y sonoras (buzzer) según sea correcta o incorrecta.
  • Mostrar mensajes de bienvenida personalizados en el LCD cuando el usuario ingresa la contraseña correcta.

MARCO TEÓRICO

Conceptos importantes

  • Arduino: Plataforma de hardware libre basada en una placa con un microcontrolador, usada para crear proyectos electrónicos.
  • Led: Dispositivo emisores de luz que se encienden al recibir corriente eléctrica en una dirección específica.
  • Resistencias: Componentes que limitan el paso de corriente eléctrica, protegiendo otros elementos del circuito como los LEDs.
  • Cables de conexión: Conductores utilizados para unir componentes electrónicos y permitir el flujo de corriente en el circuito.
  • Buzzer: Dispositivo electrónico que emite un sonido cuando recibe corriente eléctrica, usado para alarmas o alertas sonoras en proyectos electrónicos.
  • Teclado 4x4: Un teclado 4x4 es un dispositivo de entrada electrónico compuesto por 4 filas y 4 columnas, formando un total de 16 teclas. Se utiliza comúnmente en proyectos con microcontroladores (como Arduino) para ingresar números, letras o comandos. Cada tecla conecta una fila con una columna, lo que permite al microcontrolador detectar qué tecla ha sido presionada mediante un escaneo de filas y columnas..
  • Pantalla LCD: es un módulo de visualización de texto muy utilizado en proyectos con Arduino. El número "16x2" indica que puede mostrar 16 caracteres por línea en 2 líneas. Este tipo de pantalla es ideal para mostrar datos como temperaturas, distancias, mensajes, menús, entre otros

Tabla explicativa de ventajas y desventajas de usar este sistema de inicio de sesión

Ventajas Desventajas
Permite un control de acceso seguro mediante contraseña. Depende de la memoria del usuario; olvidar la clave bloquea el acceso.
El teclado 4x4 es compacto y fácil de integrar en proyectos electrónicos. El sistema puede ser vulnerable si no se implementa un cifrado o protección adicional.
La pantalla LCD proporciona retroalimentación inmediata al usuario. Requiere conocimientos de programación y manejo de periféricos para su correcto funcionamiento.
La combinación de LED y buzzer mejora la interacción y seguridad. Puede aumentar el consumo de energía si se deja activo constantemente.

PROCEDIMIENTO

Código usado en la práctica

#include LiquidCrystal.h // Librería para controlar pantalla LCD
#include Keypad.h // Librería para leer teclado matricial

// --- CONFIGURACIÓN DEL LCD ---
LiquidCrystal lcd(7, 6, 5, 4, 3, 2); // RS, E, D4, D5, D6, D7

// --- CONFIGURACIÓN DEL TECLADO ---
const byte FILAS = 4;
const byte COLUMNAS = 4;

// Mapeo de teclas
char teclas[FILAS][COLUMNAS] = {
{'1','2','3','A'},
{'4','5','6','B'},
{'7','8','9','C'},
{'*','0','#','D'}
};

byte pinesFilas[FILAS] = {A0, A1, 13, 12};
byte pinesColumnas[COLUMNAS] = {11, 10, 9, 8};

Keypad teclado = Keypad(makeKeymap(teclas), pinesFilas, pinesColumnas, FILAS, COLUMNAS);

// --- VARIABLES PARA CONTRASEÑA ---
char password[4] = "193"; // Contraseña correcta
char caracteres[4]; // Lo que ingresa el usuario
byte indice = 0; // Posición actual

// --- PINES PARA LED Y ALARMA ---
const int ledVerde = A2;
const int buzzer = A3;

void setup() {
lcd.begin(16, 2);
lcd.clear();
lcd.setCursor(0, 0);
lcd.print("Ingrese clave:");

pinMode(ledVerde, OUTPUT);
pinMode(buzzer, OUTPUT);

digitalWrite(ledVerde, LOW);
digitalWrite(buzzer, LOW);
}

void loop() {
char tecla = teclado.getKey();

if (tecla) {
lcd.setCursor(indice, 1);
lcd.print('*');
caracteres[indice] = tecla;
indice++;

if (indice == 3) { // Cuando ya ingresó los 3 dígitos
caracteres[3] = '\0';

if (!strcmp(password, caracteres)) {
lcd.clear();
lcd.setCursor(0, 0);
lcd.print("CORRECTO");
digitalWrite(ledVerde, HIGH);
digitalWrite(buzzer, LOW);
delay(2000);

// Mensaje de bienvenida
lcd.clear();
lcd.setCursor(3, 0); // Centrado (16 columnas - 9 letras) / 2 = 3
lcd.print("BIENVENIDO");
delay(3000);

// Mostrar nombre centrado
lcd.clear();
lcd.setCursor((16 - strlen("FABRICIO")) / 2, 0);
lcd.print("FABRICIO");
lcd.setCursor((16 - strlen("QUILLUPANGUI")) / 2, 1);
lcd.print("QUILLUPANGUI");
delay(3000);

} else {
lcd.clear();
lcd.setCursor(0, 0);
lcd.print("INCORRECTO");
digitalWrite(ledVerde, LOW);
digitalWrite(buzzer, HIGH);
delay(2000);
digitalWrite(buzzer, LOW);
}

// Reinicio para nueva clave
lcd.clear();
lcd.setCursor(0, 0);
lcd.print("Ingrese clave:");
indice = 0;
memset(caracteres, 0, sizeof(caracteres));
digitalWrite(ledVerde, LOW);
}
}
}

ANÁLISIS Y DISCUSIÓN

Preguntas y Respuestas - Práctica: Inicio de sesión

1. ¿Qué es un teclado 4x4?

Es un dispositivo de entrada con 4 filas y 4 columnas, formando 16 teclas, que permite ingresar datos numéricos o comandos a un microcontrolador.

2. ¿Para qué se utiliza la pantalla LCD en este sistema?

Se utiliza para mostrar mensajes al usuario, como "CORRECTO", "INCORRECTO", "BIENVENIDO" y el nombre del usuario, proporcionando retroalimentación visual.

3. ¿Qué función cumple el LED en el sistema?

Indica visualmente que la contraseña ingresada es correcta, encendiéndose cuando el acceso es autorizado.

4. ¿Qué función cumple el buzzer en el sistema?

Actúa como alarma sonora, encendiéndose cuando la contraseña ingresada es incorrecta para alertar al usuario.

5. ¿Cómo se compara la contraseña ingresada con la correcta?

Se utiliza la función strcmp() en C/C++ para comparar los caracteres ingresados con la contraseña almacenada en memoria.

6. ¿Por qué se usan asteriscos al ingresar la contraseña?

Para ocultar los números reales de la contraseña y proteger la información mientras el usuario la ingresa.

7. ¿Qué ocurre si la contraseña es correcta?

Se muestra "CORRECTO", se enciende el LED verde, luego aparece "BIENVENIDO" y finalmente el nombre del usuario en la pantalla LCD.

8. ¿Qué ocurre si la contraseña es incorrecta?

Se muestra "INCORRECTO" en el LCD, el buzzer se activa durante 2 segundos y el LED verde permanece apagado.

9. ¿Por qué se reinicia el índice y el buffer después de cada intento?

Para preparar el sistema para un nuevo ingreso de contraseña, evitando que los caracteres previos interfieran en la siguiente entrada.

10. ¿Cuál es la importancia de centrar los mensajes en la pantalla LCD?

Mejora la presentación visual, hace que los mensajes sean más legibles y profesionales, y facilita la interacción del usuario con el sistema.

Conclusiones de la Práctica

1. La integración entre sensores, pantallas y actuadores permite desarrollar sistemas interactivos y funcionales que responden a condiciones del entorno en tiempo real.
2. El uso de la pantalla LCD 16x2 facilita la visualización clara de datos medidos, mejorando la comunicación entre el sistema y el usuario.
3. Aplicar condiciones como activar un LED o un buzzer según la distancia detectada refuerza el aprendizaje de estructuras condicionales en programación.
4. Esta práctica fortalece las habilidades en electrónica básica, lógica de control y programación, permitiendo construir proyectos más complejos en el futuro.
